# Security Best Practices

This document outlines security best practices for OpenSite Analytics.

## Authentication Security

### Password Security

**Current Implementation:**
- Passwords hashed with bcrypt
- Salt automatically generated by bcrypt
- Minimum password length: 8 characters (enforced by frontend)

**Best Practices:**
- Require strong passwords (12+ characters, mixed case, numbers, symbols)
- Implement password strength meter
- Enforce password expiration (optional)
- Prevent password reuse
- Implement account lockout after failed attempts

**Implementation Example:**
```python
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain_password: str, hashed_password: str) -> bool:
    return pwd_context.verify(plain_password, hashed_password)
```

### JWT Token Security

**Current Implementation:**
- HS256 algorithm
- 30-minute token expiration
- Token stored in localStorage
- Bearer token authentication

**Best Practices:**
- Use strong secret keys (32+ random characters)
- Implement token refresh mechanism
- Store tokens in httpOnly cookies (more secure)
- Use shorter expiration times (15-30 minutes)
- Implement token revocation list
- Add token fingerprinting

**Generating Secure Secret:**
```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

**Token Refresh Flow:**
```
1. Access token expires
   ↓
2. Frontend sends refresh token
   ↓
3. Backend validates refresh token
   ↓
4. Backend issues new access token
   ↓
5. Frontend updates stored token
```

## API Security

### Rate Limiting

**Current Implementation:**
- 100 requests per minute per IP
- 200 requests per minute for tracking endpoints
- In-memory rate limiting

**Best Practices:**
- Implement distributed rate limiting (Redis)
- Use exponential backoff for violations
- Log rate limit violations
- Implement IP whitelisting for trusted sources
- Different limits for different user tiers

**Configuration:**
```env
RATE_LIMIT_REQUESTS=100
RATE_LIMIT_PERIOD=60
RATE_LIMIT_TRACKING_REQUESTS=200
```

### CORS Configuration

**Current Implementation:**
- Configured via environment variable
- JSON array of allowed origins

**Best Practices:**
- Only allow specific domains
- Validate Origin header
- Implement preflight caching
- Use strict CORS policies
- Consider wildcard only for development

**Example Configuration:**
```env
CORS_ORIGINS=["https://yourdomain.com", "https://www.yourdomain.com"]
```

### Input Validation

**Current Implementation:**
- Pydantic schemas for validation
- Type checking
- Length constraints

**Best Practices:**
- Validate all input data
- Sanitize user input
- Use parameterized queries
- Implement content-length limits
- Validate file uploads (type, size)
- Escape output to prevent XSS

**Example:**
```python
from pydantic import BaseModel, constr, EmailStr

class UserCreate(BaseModel):
    email: EmailStr
    password: constr(min_length=8, max_length=100)
    name: constr(max_length=255)
```

## Data Security

### Encryption at Rest

**Current Status:** Not implemented

**Recommended:**
- Encrypt sensitive database fields
- Use database-level encryption
- Encrypt backups
- Secure encryption keys
- Implement key rotation

**Example with SQLAlchemy:**
```python
from sqlalchemy_utils import EncryptedType
from sqlalchemy import TypeDecorator, String

class EncryptedString(TypeDecorator):
    impl = String
    cache_ok = True

    def process_bind_param(self, value, dialect):
        return encrypt(value)

    def process_result_value(self, value, dialect):
        return decrypt(value)
```

### Encryption in Transit

**Current Status:** Requires manual HTTPS setup

**Best Practices:**
- Always use HTTPS in production
- Use TLS 1.2 or higher
- Implement HSTS headers
- Use strong cipher suites
- Regular certificate renewal
- Implement certificate pinning (optional)

**Nginx Configuration:**
```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}
```

### Data Privacy

**Current Implementation:**
- IP addresses stored for analytics
- User agents stored for analytics
- No PII in default tracking

**Best Practices:**
- Anonymize IP addresses (last octet)
- Implement data retention policies
- Provide data export functionality
- Implement data deletion requests (GDPR)
- Obtain user consent for tracking
- Privacy by design

**IP Anonymization:**
```python
def anonymize_ip(ip: str) -> str:
    """Anonymize IP by setting last octet to 0"""
    parts = ip.split('.')
    parts[-1] = '0'
    return '.'.join(parts)
```

## Application Security

### SQL Injection Prevention

**Current Implementation:**
- SQLAlchemy ORM (parameterized queries)
- No raw SQL queries

**Best Practices:**
- Always use ORM or parameterized queries
- Never concatenate user input into SQL
- Use stored procedures if needed
- Implement query whitelisting
- Regular security audits

**Vulnerable Example (DON'T DO THIS):**
```python
# BAD - SQL injection vulnerability
query = f"SELECT * FROM users WHERE email = '{user_input}'"
```

**Safe Example:**
```python
# GOOD - Parameterized query
user = db.query(User).filter(User.email == user_input).first()
```

### XSS Prevention

**Current Implementation:**
- React automatic escaping
- No user-generated HTML rendering

**Best Practices:**
- Escape all user-generated content
- Use Content Security Policy (CSP)
- Implement XSS protection headers
- Sanitize HTML if needed
- Use textContent instead of innerHTML

**CSP Header:**
```http
Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'
```

### CSRF Protection

**Current Status:** Not implemented

**Recommended:**
- Implement CSRF tokens for state-changing requests
- Validate CSRF tokens on server
- Use SameSite cookie attribute
- Implement referrer checking
- Use double-submit cookie pattern

**FastAPI CSRF:**
```python
from fastapi_csrf_protect import CsrfProtect

csrf = CsrfProtect(secret_key="your-secret-key")
```

### Secret Management

**Current Implementation:** Environment variables

**Best Practices:**
- Never commit secrets to version control
- Use secret management service (HashiCorp Vault, AWS Secrets Manager)
- Rotate secrets regularly
- Use different secrets for dev/staging/prod
- Implement secret encryption at rest
- Audit secret access

**Example with Vault:**
```python

client = hvac.Client(url='https://vault.example.com')
client.auth.approle.login(role_id='role-id', secret_id='secret-id')
secret = client.secrets.kv.v2.read_secret_version(path='opensite/analytics')
```

## Incident Response

### Security Incident Response Plan

1. **Detection**
   - Monitor logs for suspicious activity
   - Set up alerts for security events
   - Regular security scans

2. **Containment**
   - Isolate affected systems
   - Revoke compromised credentials
   - Block malicious IPs

3. **Eradication**
   - Remove malware/backdoors
   - Patch vulnerabilities
   - Update systems

4. **Recovery**
   - Restore from clean backups
   - Verify system integrity
   - Monitor for recurrence

5. **Lessons Learned**
   - Document incident
   - Update procedures
   - Train team
