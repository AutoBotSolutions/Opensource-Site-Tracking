# License Compliance Guide

This guide explains how to comply with OpenSite Analytics licenses.

## MIT License Compliance

### Requirements

1. **Include Copyright Notice**
   ```html
   <!-- OpenSite Analytics - MIT License -->
   <!-- Copyright (c) 2026 OpenSite Analytics -->
   ```

2. **Include License Copy**
   - Include MIT license in documentation
   - Include license in source code comments

3. **No Warranty Disclaimer**
   - State that software is provided "as is"
   - No warranties are provided

4. **Attribution**
   - Credit OpenSite Analytics in documentation
   - Credit in product about page

### Example Attribution

```html
<footer>
  <p>Analytics powered by <a href="https://github.com/opensite/analytics">OpenSite Analytics</a></p>
  <p>Copyright (c) 2026 OpenSite Analytics. MIT License.</p>
</footer>
```

## Commercial License Compliance

### Requirements

1. **Do Not Redistribute**
   - Cannot share source code
   - Cannot distribute modified versions

2. **Keep Copyright Notices**
   - Maintain copyright in source code
   - Do not remove license headers

3. **No Source Code Modification**
   - Cannot modify source code
   - Cannot add custom features without permission

4. **Annual Renewal**
   - Renew license annually
   - Stop use if not renewed

### Verification

Licensor may request:
- Deployment verification
- Usage verification
- License key verification

## Enterprise License Compliance

### Additional Requirements

1. **Deployment Reporting**
   - Report number of deployments quarterly
   - Provide deployment details upon request

2. **Security Audits**
   - Allow security audits with 30 days notice
   - Provide access to deployment environment

3. **SLA Compliance**
   - Monitor uptime
   - Report SLA violations

### Best Practices

- Maintain deployment documentation
- Track usage metrics
- Report issues promptly
- Participate in quarterly reviews

## OEM License Compliance

### Additional Requirements

1. **Attribution**
   - Display "Powered by OpenSite Analytics"
   - Include attribution in documentation

2. **Distribution Reporting**
   - Report distributed units quarterly
   - Provide product details upon request

3. **No Standalone Distribution**
   - Must embed in another product
   - Cannot distribute as standalone analytics

### Example Attribution

```html
<div class="powered-by">
  Analytics powered by <a href="https://opensite-analytics.com">OpenSite Analytics</a>
</div>
```

## White Label License Compliance

### Additional Requirements

1. **Source Code Attribution**
   - Maintain copyright in source code
   - Acknowledge technology provider in legal agreements

2. **No Competing Products**
   - Cannot create competing analytics product
   - Cannot market as alternative to OpenSite Analytics

3. **Sales Reporting**
   - Report sales numbers quarterly
   - Provide revenue details quarterly

### Legal Agreements

Include in end-user license agreement:
```
This product includes technology from OpenSite Analytics.
OpenSite Analytics retains all rights to the underlying technology.
```

## Common Compliance Issues

### Issue: Removing Copyright Notices

**Solution:** Always maintain copyright notices in source code and documentation.

### Issue: Redistribution Without License

**Solution:** Purchase appropriate license (OEM or White Label) for redistribution.

### Issue: Not Renewing License

**Solution:** Set up renewal reminders. Stop use if not renewed.

### Issue: Exceeding License Scope

**Solution:** Contact sales to upgrade license if needed.

Licensor may conduct audits with 30 days notice. Provide:
- Deployment locations
- Usage metrics
- Distribution numbers
- Sales data (if applicable)

## Violation Consequences

### Minor Violations

- Warning notice
- 30 days to correct

### Major Violations

- License termination
- Cease and desist order
- Legal action

### Damages

- License fees for period of violation
- Legal fees
- Additional damages

## Reporting Violations

### Self-Reporting

If you discover a violation:
1. Contact legal@opensite-analytics.com
2. Explain the situation
3. Propose remediation
4. Cooperate with resolution

### Reporting Others

To report violations by others:
1. Email legal@opensite-analytics.com
2. Provide evidence
3. Confidential handling
